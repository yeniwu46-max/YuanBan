# 手机访问不了 8080 — 排查清单

## 已在本机完成的设置

- Windows 防火墙已放行 **TCP 8000、8080、8888**（专用 + 公用配置）
- WLAN 已设为 **专用网络**
- H5 与 API 合并为 **单端口 8080**（`/api` 由本机代理到 8000）

## 请用手机浏览器打开（二选一）

### A. 同一 WiFi（局域网）

```
http://112.81.166.217:8080/
```

手机必须：

1. 连接与电脑 **相同的 WiFi**（不要只用 4G/5G）
2. 在系统设置里查看手机 IP，应为 `112.81.166.x`（与电脑同网段）
3. 关闭「专用 WLAN 助理 / 自动切换移动数据」

若仍打不开，多半是路由器 **AP 隔离**（禁止设备互访）→ 用方案 B。

### B. 手机开热点（最稳）

1. 手机打开 **个人热点**
2. 电脑 WiFi 连接到该热点（不要连原来的校园/家里 WiFi）
3. 在项目根目录重新执行：

```powershell
.\scripts\deploy-no-docker.ps1 -UseApi
```

4. 看脚本输出的新 IP（常见 `192.168.43.x` 或 `192.168.137.x`）
5. 手机浏览器访问 `http://<新IP>:8080/`

### C. 公网隧道（已尝试 localtunnel）

若终端里打印了 `https://xxxx.loca.lt`，手机用流量也能打开（首次可能要点确认页）。

## 本机自测

电脑浏览器先打开：http://127.0.0.1:8080/  
再打开：http://127.0.0.1:8080/health 应显示 `status: ok`
